# 바이럴 런치 시퀀스

## 사전 체크리스트 (전부 통과 전 게시 금지)
- [ ] 공개 리포 README 영문 정비 (데모 GIF, quickstart, 아키텍처, honest FAQ)
- [ ] 랜딩 부하 내성: 미니 ATANOR는 브라우저 로컬이라 서버 부하 무관 — waitlist 엔드포인트만 확인
- [ ] waitlist 수집 동작 + 카운트 확인 경로
- [ ] Vercel analytics 활성 (유입 측정 없이 게시 금지)
- [ ] 데모 완주 시나리오 3개 (첫 방문자가 30초 안에 "오" 하는 경로)
- [ ] 라이선스/민감정보 스캔 (공개 리포에 키·개인정보 없음)

## 순서 (각 단계는 운영자 승인·게시)
1. **D-3**: README 개선 머지, 리포 정리. X에 빌드-인-퍼블릭 티저 1개.
2. **D-1**: 지인/커뮤니티 소프트 테스트 (피드백으로 데모 시나리오 손질).
3. **D-day (화–목)**: Show HN (KST 21–23시) → 첫 댓글 기술 해설 → 4시간 성실 응답.
4. **D+1**: r/LocalLLaMA. HN 반응 좋았으면 그 스레드 링크 포함.
5. **D+2**: X 스레드 (HN/Reddit 사회적 증거 포함), LinkedIn.
6. **D+3~7**: r/MachineLearning [P], r/selfhosted. 유입 지표 리뷰 → 반복.

## 실패 대응
- HN 무반응(<10pt): 1주 후 다른 앵글("no-LLM 한국어 엔진" vs "브라우저 로컬 AI" vs "P2P 그래프")로 1회 재시도.
- 악성 피드백: 기술 비판은 선물 — 정직하게 인정+수리 계획 답변. 감정 소모전 금지.
