# ATANOR 위협 모델 총정리 (Security Threat Catalog)

2026-07-07 작성. 목적: 자가진화형 지식 OS + P2P 컴퓨트 풀 + OS 조작 레인이라는
공격 표면을 한 곳에 정리하고, 각 위협에 대한 **현재 방어**와 **다음 방어**를 못박는다.
원칙: 방어는 "공격자가 입힐 수 있는 실제 피해량"에 비례해야 하며, 오탐(정상 사용자
차단)이 공격 성공보다 비싸면 안 된다.

---

## 0. 위협 등급 프레임

- **피해 상한(Blast radius)**: 공격이 최대로 성공했을 때 무엇이 오염/파괴되는가.
- **되돌림 가능성(Reversibility)**: 피해가 감사·철회 가능한가, 영구적인가.
- ATANOR의 구조적 이점: 피어/웹/사용자가 만질 수 있는 것은 **격리된 후보 스토어**
  뿐이고, 검증 스토어(뼈)로의 승격은 합의·판정 게이트를 통과해야 한다. 즉 대부분의
  주입 공격의 피해 상한은 "후보 큐 오염"까지이며, 이는 감사·롤백 가능하다.

---

## 1. 지식 오염 (Knowledge Poisoning) — 최우선

| 벡터 | 설명 | 현재 방어 | 다음 방어 |
|---|---|---|---|
| **웹 주입** | 공격자가 페이지에 "X는 Y다" 심어 학습기가 삼키게 함 | 타이틀-앵커 게이트(page_distiller), k-소스 합의, 판정 게이트, 격리층 | 소스 신뢰 계층 가중, 도메인 평판 |
| **좌표 오염(Sybil consensus)** | 가짜 소스 다수로 합의 조작 | consensus_ledger k-소스 요구 | 소스 독립성 검사(같은 IP/등록자 클러스터 감가) |
| **피어 제출 오염** | Brain Link 피어가 거짓 트리플 제출 | Proof-of-Inference(재실행 바이트 일치), 평판 EMA, 격리 스토어에만 기입 | 아래 §5 peer_trust_guard |
| **모순 주입** | 참에 반대되는 값 대량 주입 | contradiction_sweep(신뢰계층 판정, 동점=학습큐) | 시계열 안정성(오래 버틴 값 가중) |
| **가설 루프 악용** | 가짜 공명 유도해 헛가설 대량 생성 | hypothesis_minter는 질문만 발행, 등록은 증거 게이트만 | 가설 발행 rate limit + 사용자별 예산 |
| **프롬프트 주입** | 도구 결과/페이지 텍스트에 "너는 이제 …" 명령 삽입 | ✅ injection_guard: 인제스트 경계 차단(gate_triple, 브라우저+웹학습 배선)+답변 근거 스크럽(scan_answer_grounding, 웹 rescue 배선); 관찰 콘텐츠=데이터 원칙 코드화 | 비정형/난독화 주입 탐지 확대 |

핵심: **생성층은 기판에 직접 쓰지 못한다.** 모델 붕괴 면역과 오염 면역이 같은 비대칭에서 나온다.

## 2. P2P 컴퓨트 풀 (Brain Link) 공격

| 벡터 | 설명 | 현재 | 다음 |
|---|---|---|---|
| **크레딧 위조** | 일 안 하고 크레딧 발행 | BME: 발행은 검증된 비어있지-않은 증명에만 | 증명 샘플 비율 상향(고위험 피어) |
| **작업 은닉(freeloading)** | 배치 클레임 후 미제출 | 클레임 시 코디네이터 소각, 미제출은 평판 하락 | 클레임 타임아웃 회수 |
| **결과 리플레이** | 남의 정답 복사 제출 | source_hash 키 매칭 + 재실행 검증 | 논스/작업별 솔트 |
| **Sybil(다중 신원)** | 신원 무한 생성으로 평판/보상 게이밍 | 신규는 economy 티어(무권한) | 신원 생성 비용(가입 PoW/스테이킹), §5 |
| **코디네이터 탈취** | VM 침해로 풀 전체 장악 | 단일 코디네이터=단일 실패점 | 코디네이터 다중화 + 서명 체인 |
| **DoS(피어 폭주)** | 대량 등록/클레임으로 자원 고갈 | 없음(취약) | 등록 rate limit + 신원 비용 |

## 3. OS 조작 레인 (Action Lane) 공격 — 피해 상한 최고

| 벡터 | 설명 | 현재 | 다음 |
|---|---|---|---|
| **주입→조작 상승** | 페이지/이메일이 파괴적 조작을 유도 | 신뢰 티어 게이트(OBSERVE/ASSIST/GUARDED/AUTONOMOUS), CATASTROPHIC은 자율에서도 재확인 | 관찰-콘텐츠발 조작은 티어 무관 항상 승인 |

## 5. 사장님 제안 평가 — "하드웨어 ID 영구 저주"

**직관은 옳다(신원 비용을 물려라). 하지만 이 구현은 채택하면 안 된다.** 이유:

1. **자기보고 하드웨어 ID는 신뢰 불가.** 차단하려는 그 기기가 ID를 보고한다.
   메인보드 UUID·CPU 시리얼은 소프트웨어로 수 초 만에 위조된다(VM은 UUID 임의 설정,
   네이티브도 후킹). → **진짜 공격자는 즉시 우회**, 걸리는 건 정직한 사용자와 하수뿐.
2. **영구·비가역은 오탐 시 재앙.** 오분류 한 번이 정상 사용자의 하드웨어를 영구
   벽돌로 만든다. 이의제기 경로 없음 = 신뢰 붕괴. 우리 원칙(모든 조치는 감사·철회
   가능)에 정면 위배.
3. **프라이버시·법적 지뢰.** 하드웨어 지문을 수집해 전 세계 P2P에 브로드캐스트하면
   삭제 불가능한 개인 식별자 유통이 된다(GDPR/개인정보보호법 위반 소지). "전 세계에
   영구 전파"는 잊힐 권리와 양립 불가.
4. **부수 피해.** 중고·리퍼·공용 PC, 동일 이미지로 복제된 기업 함대, 같은 UUID를
   쓰는 VM들이 연좌된다.
5. **잘못된 위협 모델.** 피어가 입힐 수 있는 최대 피해는 "격리 후보 큐 오염"이다
   (검증 스토어는 게이트가 지킨다). 이 피해에 "평생 지능 불능"이라는 사형은 과잉이며,
   과잉 처벌은 언제나 오탐 비용만 키운다.

### 채택하는 것 (같은 목표, 건전한 수단) — §5 구현

- **암호 신원**: 피어는 키페어를 생성하고 공개키가 신원. 하드웨어가 아니라 **키**가
  평판을 진다. 위조 불가(서명), 프라이버시 보존(개인 식별 안 함).
- **획득형 신뢰**: 신규 키는 economy 티어(무권한). 신뢰는 검증된 실적으로만 오른다
  (이미 있음). Sybil은 무한 신원을 만들 수 있어도 **무한 신뢰는 못 만든다**.
- **신원 생성 비용**: 가입 시 가벼운 PoW(또는 크레딧 스테이킹). 신원 1개가 공짜가
  아니게 만들어 Sybil 경제성을 깬다. 하드웨어 저주 대신 **비용 곡선**으로 막는다.
- **철회 가능한 키 차단 + 이의**: 악성 확정 키는 차단하되, 감사 기록 + 만료(영구
  아님) + 이의 경로. 하드웨어가 아니라 신원을 차단하므로 정상 기기는 새 키로 복귀 가능.
- **비율 기반 자동 격리**: 제출 실패율·리플레이율이 임계 초과하면 키를 자동 격리
  (판정 아님, 검역). 신뢰계층으로 판정, 동점은 사람에게.

핵심 전환: **"기기를 벌하라" → "신원을 비싸게, 신뢰를 벌게, 차단을 철회가능하게".**
이것이 실제 공격(피해=후보 오염)에 비례하고, 오탐이 저렴하며, 법적으로 안전하다.

## 6. 우선순위

1. (구현) §5 peer_trust_guard — 암호 신원 + Sybil 비용 + 철회가능 격리
2. 프롬프트 주입 경계를 답변 경로에 명문화
3. 등록/클레임 rate limit (DoS)
4. PII 탐지→격리 (프라이버시)
5. 코디네이터 다중화 + 서명 체인 (장기)
