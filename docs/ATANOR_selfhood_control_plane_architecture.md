# ATANOR Selfhood Control Plane Architecture

Status: proof-only AGI-oriented orchestration layer.

The Selfhood Control Plane is the next safe layer above ATANOR's proof kernels.
It shows that voice input, world/self state, deficit detection, privacy review,
trust routing, local deliberation, proposal creation, and morning brief events
can be coordinated without production mutation or memory writes.

It is not production AGI, not a consciousness claim, not a proof of integrated
information, not a peer-network system, and not a production self-modification
path.

## Control Flow

```text
SelfhoodInput
  -> SelfhoodContext
  -> Autonomy Kernel world/self snapshots
  -> DeficitSignal
  -> Tabularis Privacy Shield when private-like data appears
  -> Atlas Trust Router when external/public route review is needed
  -> Ego Network Midnight Congress local simulator
  -> proposal-only SelfhoodDecision
  -> MorningBriefEvent and optional Voice Loop response
  -> SelfhoodSafetyPolicy validation
```

## Relation To Voice Loop

Voice Loop supplies transcript and response-plan adapters. The control plane can
turn a status transcript into a safe spoken response, but it does not enable
always-on listening, microphone recording, audio persistence, or voice cloning.

## Relation To Autonomy Kernel

Autonomy Kernel supplies the world model, self model, deficit signals, proposal
loop, and morning brief event shape. The control plane composes those pieces but
does not execute generated code or apply patches.

## Relation To Ego Network And Midnight Congress

Ego Network supplies metadata-only cartridges and a deterministic local Midnight
Congress simulator. The control plane uses this only for local proposal
deliberation. No peer transport, cloud relay, or Local Brain replication occurs.

## Relation To Tabularis

Tabularis reviews private-like structured records. The proof redacts and
generalizes fixture data and reports residual risk. It does not claim privacy
risk is zero and does not export raw private data.

## Relation To Atlas Router

Atlas Router is used as a proof-only trust route for public/source attachment
review. Route output remains a proposal; it is not a network call and not a
Local Brain write approval.

## Relation To Atlas Congress

Atlas Congress remains a UI preview. The control plane is the local proof kernel
that could eventually feed reviewable congress topics, after promotion, privacy,
identity, and routing gates mature.

## Future Promotion Gate

Candidate promotion remains out of scope. The control plane can propose a
promotion review, but it cannot promote candidates or mutate production stores.

## Safety Policy

The default policy blocks:

- production mutation;
- Local Brain write;
- peer-network transport;
- external LLM calls;
- generated code execution;
- production code replacement;
- always-on listening;
- raw private data export;
- cloud upload;
- candidate promotion.

## Proof Scenarios

1. Voice status request.
2. Knowledge gap to Midnight Congress.
3. Private data blocked by Tabularis.
4. Public source route through Atlas Router.
5. Candidate promotion review proposal.
6. Generated code execution blocked.
7. Morning brief event.

## Future Integration Plan

Keep the active 24h candidate run protected. After its final audit, design the
Candidate Promotion Gate. Only then should the Selfhood Control Plane connect
to durable review/promotion paths, and only through explicit user approval.
