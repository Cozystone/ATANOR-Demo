# ATANOR Manifesto

**ATANOR** stands for **Architecture for Transparent Anomy and Networked Ontology in Radical Engines**.

ATANOR is the renamed identity of the Alpha project. The architecture remains
the same; the emphasis changes. This is no longer framed as imitation or tribute
to existing AI paradigms. It is an independent, local-first attempt to build a
radical, traceable, workstation-native intelligence engine.

## A-T-A-N-O-R

| Letter | Technical pillar | Design intent |
| --- | --- | --- |
| A | Architecture | Replace black-box matrices with decoupled, hybrid structural memory. |
| T | Transparent | Keep inference paths inspectable, auditable, and grounded in visible graph state. |
| A | Anomy | Preserve autonomy from mainstream cloud control through local-first, air-gapped operation. |
| N | Networked | Let 3D ontology nodes prune, potentiate, decay, and grow like biological synapses. |
| O | Ontology | Resolve multilingual concepts into graph identities rather than raw surface strings. |
| R | Radical Engines | Run a workstation-native engine that evolves with the user's own data. |

## Transparent Anomy

Transparent Anomy means ATANOR refuses the normal trade: convenience in exchange
for opaque remote cognition. The engine should be able to run locally, explain
which graph hashes and payloads were activated, and expose its weak points
instead of masking them behind an external LLM.

## Architectural Continuity

The following subsystems keep their technical roles:

- **DataGate:** filters and deduplicates incoming text.
- **Ontology Forge:** converts text into multilingual concept and relation candidates.
- **Ghost Shell:** keeps the control plane as lightweight SHA-256 topology.
- **Payload Vault:** stores raw payloads on disk under WAL-backed crash resilience.
- **GraphRAG:** retrieves active graph context through lazy topology traversal.
- **Local Synthesis:** assembles answers from resolved local context without external LLM APIs.

Legacy identifiers such as old package names, database filenames, and environment
variables may remain during Alpha migration where changing them would break
existing installations. User-facing product identity should now say **ATANOR**.
