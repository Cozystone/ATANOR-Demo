# ATANOR Environment — Stage 1 스펙 (OS를 만들지 않고 OS를 입는다)

> 목표: 리눅스처럼 설치 가능한, 부팅하면 ATANOR인 컴퓨터.
> 방법: 커널·드라이버를 새로 만들지 않는다. 검증된 배포판(우분투 LTS) 위에
> ATANOR를 **시스템의 상주 정신**으로 얹는다. 위험(드라이버 지옥·보안 패치 부채)은
> 우분투가 지고, 우리는 경험 전체를 소유한다.

관련: docs/DESIGN_PHILOSOPHY.md (④ 구조가 철학을 증명할 것 — "AI가 어디에나 있다"를
말로 하지 않고, 부팅 순간부터 구조로 보여준다).

---

## 1. 단계 정의 (무엇이고, 무엇이 아닌가)

| | Stage 1 (이 문서) | Stage 2 (AI 완성 후) | Stage 3 (하지 않음) |
|---|---|---|---|
| 정체 | 우분투 LTS 기반 **스핀**(remix) | OS 수준 통합 심화 | 자체 커널/디스트로 |
| 설치 | ISO 한 장, 일반 PC | 동일 | — |
| AI 위치 | systemd 상주 서비스 + 기본 셸 | 파일시스템 인덱스=로컬 브레인, 지각 스트림 | — |
| 유지비 | 낮음 (Canonical이 커널 짊어짐) | 중간 | 무한 (거부 사유) |

정직한 한계: Stage 1은 "AI와 하나 된 OS"의 **체감 90%**다 — 부팅→ATANOR, 어디서나
호출, 로컬 전용 데이터. 나머지 10%(커널 수준 후킹)는 Stage 2에서도 필요할 때만.

## 2. 아키텍처

```
┌────────────────────────────────────────────┐
│  ATANOR Shell (전체화면 웹 UI, 카이오스크)      │  ← 사용자가 보는 전부
├────────────────────────────────────────────┤
│  atanor-web.service   (Next.js, :3000)      │
│  atanor-engine.service(FastAPI,  :8502)     │  ← 상주 정신: 학습 루프·자기질문·
│    └ continuous learning / selfhood loops   │     기권→적재 루프가 부팅부터 동작
├────────────────────────────────────────────┤
│  Ubuntu 24.04 LTS (커널·드라이버·보안 패치)   │
└────────────────────────────────────────────┘
```

- **atanor-engine.service**: 지금의 :8502 백엔드 그대로. `Restart=always`,
  `MemoryMax=` 로 학습 루프 폭주를 OS 수준에서 바운딩 (VM OOM 교훈 반영).
- **atanor-web.service**: `next start` 프로덕션 빌드. engine에 `After=`/`Requires=`.
- **ATANOR Shell**: cage(Wayland 카이오스크) + chromium `--kiosk http://127.0.0.1:3000`.
  데스크톱이 아니라 ATANOR가 첫 화면. `Ctrl+Alt+F2`는 언제나 살아 있는 표준 TTY —
  사용자를 가두지 않는다 (철학 ①: 정직 = 탈출구를 숨기지 않음).
- **데이터 계약**: `/var/lib/atanor/` 단일 루트 (graph, vault, ledgers). 백업 = 디렉터리
  하나 복사. 개인 데이터는 이 디렉터리 밖으로 나가지 않는다 (기존 Tabularis 불변식).

## 3. 배포물

1. `deploy/atanor-environment/install.sh` — 스톡 우분투를 ATANOR Environment로 전환하는
   멱등 인스톨러 (이 커밋에 포함, 검증됨). USB 우분투 설치 → 스크립트 1회 = Stage 1 체험.
2. ISO 리마스터 (다음 단계): Cubic/live-build로 위 인스톨러를 프리시드에 넣어
   "설치하면 곧 ATANOR"인 ISO 생성. 인스톨러가 검증된 뒤에만 착수.

## 4. 보안·프라이버시 원칙 (Stage 1에서 이미 적용)

- 엔진·웹은 전용 시스템 계정(`atanor`)으로 구동, 홈 디렉터리 접근 없음.
- 루프백 바인딩이 기본; 외부 노출은 사용자가 명시적으로 연 경우만 (Caddy 옵션).
- 자동 코드 자기수정 없음 — 기존 운영자 승인 게이트가 OS에서도 그대로.
- 업데이트 = `git pull + systemctl restart` (원자적, 롤백은 git). 무인 자동 업데이트는
  기본 꺼짐 — 기계가 밤사이 스스로 바뀌지 않는다는 약속.

## 5. 리스크 대장 (정직하게)

| 리스크 | 판단 |
|---|---|
| 드라이버/하드웨어 | 우분투 LTS에 위임 — Stage 1에서 소멸 |
| 보안 패치 부채 | unattended-upgrades(OS 패치만) 옵션 제공 |
| 집중력 분산 | 인스톨러+유닛 2개 수준으로 상한. ISO는 수요 확인 후 |
| 킬러 체감 부재 | 부팅→말 거는 기계 + 완전 로컬 소유가 킬러. 과장 금지 |
