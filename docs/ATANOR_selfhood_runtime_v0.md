# ATANOR Selfhood Runtime v0

Selfhood Runtime v0 is ATANOR's first integrated proof-only self-model loop. It
does not add a new production autonomy path. It connects existing proof axes so
ATANOR can observe a local input, detect a deficit, deliberate, check safety
gates, propose an action, and answer in text or optional voice while waiting for
explicit user approval.

This is not real consciousness, not AGI, not an IIT proof, and not automatic
self-modification.

## Connected Axes

- Autonomy Kernel: world/self snapshot and deficit signal context
- Ego Network: identity and Midnight Congress architecture context
- Digital Life Kernel: safe action proposal model
- MiroFish: deterministic local deliberation roles
- Candidate Promotion Gate: dry-run-only candidate review
- Tabularis Privacy Shield: direct identifier and private-data risk checks
- Atlas Router / P2P Sandbox: dry-run routing, no real peer connection
- Voice Loop: voice transcript support and optional mock/fallback voice output
- Logical Sphere: read-only verified/candidate/rendered count semantics

## Unified Text and Voice Input

Text remains the primary input path. Voice transcript input enters the same
runtime bus as text and does not enable an always-on microphone. Optional voice
output is represented as a mock/fallback event and does not persist generated
audio.

## Approval-required Action Model

The runtime produces proposals, not actions. Nontrivial proposals require user
approval. The proof runtime blocks or downgrades requests that would mutate
production, write Local Brain, perform real promotion, connect real P2P, upload
to cloud, execute generated code, perform hot-swap, or claim real consciousness
or achieved AGI.

## What v0 Does Not Do

- no production `verified_store_v0` mutation
- no Local Brain write
- no candidate store mutation
- no real promotion manifest
- no real P2P
- no real cloud upload
- no external LLM or sLLM call
- no generated code execution
- no UI or API integration
- no always-on microphone
- no voice cloning
