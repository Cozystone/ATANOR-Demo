# ATANOR 최종 로드맵 — "마법 같은 AI OS"까지

Owner 비전 (2026-07-06): 바탕화면에 ATANOR 파티클이 상주하고, 자비스처럼 말로 모든
조작·자동화가 가능하며, 파티클이 컴퓨터를 자기 집처럼 이동하고(창이 뜨면 비켜 앉는 등),
컴퓨터–스마트폰–스마트글래스가 하나로 연결되어 AI가 기기 사이를 옮겨 다니며 사용자의
모든 맥락을 이해하는 OS. 이것이 궁극적 방향성이다.

## 배포 형태 두 가지 (exe vs OS)

- **exe(앱)**: 지금 쓰는 Windows/맥 위에 설치하는 프로그램. 사용자의 기존 환경을
  건드리지 않고 ATANOR만 얹는다. 보급용 — 진입장벽이 없다.
- **OS(ATANOR Environment)**: 컴퓨터를 켜면 ATANOR가 곧 그 컴퓨터인 상태. 우리가
  화면·마이크·파일·창 관리까지 소유하므로 "파티클이 집처럼 산다"는 비전은 OS에서만
  완전해진다. ISO 설치 → 첫 부팅 자동 프로비저닝까지 QEMU 부팅 검증 통과(2026-07-06).
- 전략: exe로 보급하고, OS로 궁극 경험을 완성한다. 같은 엔진, 두 껍데기.

## 난제 (정직한 목록)

1. **답변 품질 절대치** — GPT-4 수준 grounded QA까지 커버리지·정확도(현 스냅샷 ~45%).
   관계 추출 노이즈, 팩 승격 게이트, 웹→그래프 루프 품질이 병목.
2. **음성 실시간성** — No-LLM 로컬 STT/TTS(Whisper류는 LLM 아님, 사용 가능)와 의도
   해석을 그래프 라우팅으로. 발화→행동 1초 내.
3. **OS 조작의 안전 게이트** — "말로 모든 조작"은 파괴적 행동 승인 모델(지금의
   operator_only 철학을 UI로)과 실행 취소 원장이 선행되어야 한다.
4. **지각 스트림** — 화면/입력의 연속 관찰을 로컬에서만, 유출 없이. 저장은 그래프
   요약(개념·관계)으로, 원본은 버린다. 프라이버시가 곧 아키텍처인 설계.
5. **기기 간 연속성** — Brain Link P2P를 신원·암호화·핸드오프(대화 상태 이동)로 확장.
6. **조단위 그래프 상시 운용** — int-columnar 스토어는 있음; 상시 서빙·증분 학습·
   경량 기기(폰/글래스)용 슬라이스가 남음.

## 단계별 성장 목표

- **S1. 지금** — 데모 완성도: 랜딩/데모/클라우드/AGORA + exe 패키징, 답변 품질 스윕.
- **S2. 개인화 완성** — 로컬 브레인이 사용자의 파일·습관·대화를 그래프로 누적, "나를
  아는 AI"를 측정 가능하게 (개인 맥락 재현률 배터리).
- **S3. ATANOR Environment 1.0** — 지금의 ISO에 파티클 셸(바탕화면 상주 오브),
  음성 명령 1차(앱 실행/검색/받아쓰기), 모든 행동에 승인 게이트.
- **S4. 자비스 단계** — 지각 스트림(로컬 전용) + 창 관리 통합(파티클이 창과 상호작용),
  자연어 자동화(반복 작업을 그래프 절차로 학습).
- **S5. 연속성** — 폰 컴패니언 → 글래스. 기기 간 핸드오프, 하나의 자아가 이동.
