# ATANOR Distributed Compute Vision

ATANOR's long-term network direction is a local-first, cloud-assisted compute fabric. The Alpha build remains a sovereign local engine first; distributed compute is prepared as architecture, not falsely advertised as live infrastructure.

## Non-Negotiable Constraints

- No arbitrary remote code execution.
- No private Local Brain data sharing.
- No Payload Vault upload by default.
- No blockchain or token-transfer mechanism in Alpha.
- No dependency on external LLM APIs for answer generation.

## Intended Evolution

### Stage 0: Sovereign Local Engine

The user's workstation runs ingestion, Ghost Shell topology, Payload Vault storage, lazy GraphRAG retrieval, and local synthesis. This is the current Alpha baseline.

### Stage 1: Contributor Node Preview

The app can register a local node and process public-only verification jobs from a local preview broker. This validates the UX, privacy boundary, task schema, and resource caps.

### Stage 2: Cloud Brain Broker

A production broker may later assign signed public tasks to opted-in Contributor Nodes. The broker should exchange metadata and public fragments only.

### Stage 3: Federated Edge Verification

Multiple nodes can independently validate public fragments. The network can compare result hashes and confidence signals before accepting a public graph update.

## Internal Credit

Contribution Credit is an Alpha accounting signal for work performed. It is intentionally non-financial in this version.

## Relationship To Local Learning

Local Learning improves the user's private Local Brain over time. Contribution helps public Cloud Brain quality only when explicitly enabled. These two paths must stay visually and architecturally separate.
