## 고질 문제 클래스 6종과 해법

### 1. 오참조 (wrong-referent) — ✅ 이번에 뿌리 수정
증상(반복 이력): 팔란티어→Miraculous(웹), 방탄소년단→**탄소**, 삼성전자→**전자**(팩).
뿌리: `_concept_score`의 원시 부분문자열 `name in query` +2.2점. 지금까지는 표면(웹 구제 게이트,
referent-type resonance)마다 개별 패치 — 원점은 방치돼 있었음.
**혁신**: 패치를 경로마다 붙이지 않고 **채점 프리미티브 자체에 언어학적 최대 일치 원칙** 적용
(`_named_with_boundary`: 좌측 한글=단어 내부 → 거부, 우측 한글 런이 조사가 아니면 더 긴 용어 → 거부).
스캔·sqlite 인덱스 두 경로가 동시에 고쳐짐. 라이브 재측정으로 환각 14%→10% 확인.

### 2. 학습 추출 노이즈 → 그래프 내 오사실 (현재 최대 잔여 리스크)
증상: 퀴리→"핵분열 발견"(실제는 한·슈트라스만), 중력 is_a 이론(오접목). 합의-증거 기계·정의문
게이트가 있어도 소량 누수.
**혁신 (설계)**: **큐레이션 트리플 스토어를 심판으로 쓰는 모순 탐지** —
(a) 학습된 관계 (s,p,o)를 승격 전에 큐레이션 KG(Wikidata 등급 소스)와 대조, 동일 (s,p)에 다른 o가
큐레이션에 있으면 자동 격리; (b) 기존 진실발견 EM에 큐레이션 트리플을 고신뢰 앵커 소스로 배선;
(c) 파생 대수로 **타입 위반 탐지**(중력의 is_a 부모가 '상호작용'과 '이론'처럼 서로소 타입이면 플래그).
새 크롤이 아니라 이미 가진 두 자산(합의 기계 + 트리플 스토어)의 결합이라 저비용.

### 3. write-store ≠ read-store (학습이 답변에 안 닿음)
반복 이력: cloud brain 형제 스토어, 팩-빌더 동결 소스, 그래프↔팩 분리. 매번 증상 발견 후 수동 정렬.
**혁신 (설계)**: **스토어 계약 레지스트리** — 각 컴포넌트가 (이름, write_path, read_path)를 등록하고
CI 테스트가 일치를 검증 + 답변 인증서에 read-store 지문, 학습기에 write-store 지문을 넣어
런타임 워치독이 불일치 시 경보. "학습했는데 왜 몰라?"류를 클래스로 차단.

### 4. 커버리지 vs 기권 (홀드아웃 32%)
정직한 기권은 옳지만 성장축은 커버리지. **해법은 이번 스케일 기판이 정확히 그것**: 큐레이션 벌크
적재(584k/s) + 코퍼스 어댑터(Tatoeba/Wiktionary/OSCAR/AI-Hub) + 파생 추론(×3) + 자동 팩 승격.
홀드아웃 기권 용어 목록(성남시, Roblox, 장관급…)이 그대로 다음 적재 타깃 리스트가 된다 —
**기권 로그를 적재 큐로 되먹이는 루프**가 다음 배선.

### 5. 조사/형태 표면 버그 (받침·조사·이/가)
반복 소액 버그. 현재 `_ko_topic`/조사 셋/으로·로 규칙이 각 모듈에 분산.
**해법**: LAD 형태층 단일 모듈로 통합(지식 아님, 코드 허용 층). 신규 표면 코드는 그 모듈만 쓰도록.
