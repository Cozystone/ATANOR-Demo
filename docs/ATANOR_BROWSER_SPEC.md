# ATANOR Browser — 3박자 배포의 중간 고리

Owner 비전 (2026-07-06): AI 앱 / 브라우저 / OS 3형태 배포. 각각 단독 작동, 합치면 완전체.
브라우저의 존재 이유 = **전방위 인터넷 활동의 원자적 흡입 → 초개인화 컨텍스트 장부**,
사유·취향의 동기화, 그리고 사생활이 빅테크 서버로 절대 넘어가지 않는 구조.

## 정직한 엔진 판단

브라우저 "엔진"(렌더링)을 처음부터 만드는 것은 수백 엔지니어-년 규모 — 하지 않는다.
Brave/Arc/Edge가 그렇듯 **Chromium 오픈소스 엔진 위의 자체 브라우저 제품**이 정답이다.
구글과의 협업이 아니라 오픈소스 사용이므로 허락이 필요 없다. 우리가 만드는 것은:
셸(UI) + ATANOR 통합(사이드바 오브/음성) + 흡입 파이프라인 + 로컬 장부.

## 배포 사다리 (각 단계가 단독 제품)

0. **확장 프로그램** (진입장벽 최저, 크롬/엣지 사용자 그대로): 흡입 파이프라인만 —
   방문/체류/선택 텍스트 → 로컬 엔진(:8502)으로 개념 이벤트 전송. 브라우저 전환 없이
   컨텍스트 장부가 시작된다. 기존 플러그인 킷과 접점.
1. **AI 앱** (배포됨: ATANOR_0.1.2_x64-setup.exe) — 엔진+대시보드.
2. **ATANOR Browser** — Chromium 포크: 오브가 신탭/사이드바에 상주, 모든 페이지가
   원자 이벤트로 흡입, 로컬 장부가 곧 개인화 레이어. 검색/추천이 그래프에서 나온다.
3. **ATANOR OS** (부팅 검증 완료) — 데스크톱 전체가 같은 원리로.

## 원자적 흡입 (Atomic Ingestion) 계약

- 단위: (URL 도메인, 개념들, 관계들, 체류/스크롤 신호, 시각) — **원문 HTML은 저장하지
  않는다. 파기가 기본.** 그래프 요약만 로컬 장부에 남는다 (지각 스트림과 동일 원칙).
- 저장소: Local Brain (triple store + 개인 컨텍스트 장부). 클라우드 업로드 금지가
  아키텍처 (전송 코드 자체가 없음 — 정책이 아니라 구조).
- 동기화: 기기 간은 Brain Link P2P 암호화 직결로만. 서버 경유 없음.
- 사유·취향 동기화: 장부에서 파생된 선호/관심 그래프가 답변 라우팅과 추천의 prior가
  된다 (경험 원장과 같은 방식의 측정 기반 라벨).

## 순서

확장(0)이 먼저다: 브라우저 포크 없이 흡입 계약을 실증하고, 그 파이프라인을 그대로
포크에 이식한다. 포크 빌드는 CI 인프라(수시간 빌드)가 필요하므로 확장으로 가치를
증명한 뒤 착수한다.
